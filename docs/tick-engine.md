# Tick Engine

The ripple-wave processor — the system's heartbeat. The tick engine is the Mediator pattern — primitives never communicate directly. They emit events, and the tick engine routes those events to subscribing primitives in subsequent waves.

## Concepts

### Tick

A `Tick` is a single invocation of the engine. Each tick processes all pending events through all eligible primitives, potentially generating new events that trigger further processing within the same tick (ripple waves).

### Wave

A wave is a single pass through eligible primitives within a tick. Wave 0 processes externally-generated events. Waves 1+ process events generated by previous waves. A tick contains 1 to `MaxWavesPerTick` waves (default: 10).

### Quiescence

A tick reaches quiescence when a wave produces no new events. This is the normal termination condition. The wave limit (`GraphConfig.MaxWavesPerTick`) is a safety valve — if the system hasn't quiesced after 10 waves, the tick ends and remaining events carry over to the next tick.

## Tick Algorithm

```
fn RunTick(tick Tick, store Store, bus IBus, registry PrimitiveRegistry, config GraphConfig) → Result<TickResult, StoreError>:

    // 1. Snapshot — capture all primitive states as a frozen, read-only view
    snapshot = Frozen(Snapshot {
        Tick:          tick,
        Primitives:    registry.AllStates(),
        PendingEvents: bus.Drain(),              // events since last tick
        RecentEvents:  store.Recent(100, None),  // context window
        ActiveActors:  actorStore.List(ActorFilter { Status: Some(Active) }),
    })

    pendingEvents = snapshot.PendingEvents
    allMutations = []
    waveCount = 0

    // 2. Ripple-wave loop
    while waveCount < config.MaxWavesPerTick:
        waveMutations = RunWave(tick, waveCount, pendingEvents, snapshot, registry)

        if waveMutations is empty:
            break   // quiescence — no new mutations means no new events

        allMutations.append(waveMutations)

        // Extract new events from mutations for next wave
        pendingEvents = waveMutations
            .filter(m → m is AddEvent)
            .map(m → m.toEvent())

        if pendingEvents is empty:
            break   // quiescence — mutations exist but no new events

        // Update snapshot with new events for next wave
        snapshot = snapshot.withPendingEvents(pendingEvents)
        waveCount += 1

    // 3. Apply all mutations atomically
    ApplyMutations(allMutations, store, bus, registry)

    // 4. Persist primitive states
    registry.PersistStates()

    // 5. Emit tick completion event
    store.Append(EventFactory.Create(
        type:    MustEventType("clock.tick"),
        content: ClockTickContent { Tick: tick, Waves: waveCount, ... },
        ...
    ))

    return TickResult { Tick: tick, Waves: waveCount, Mutations: len(allMutations) }
```

## Wave Algorithm

```
fn RunWave(tick Tick, wave int, events []Event, snapshot Frozen<Snapshot>, registry PrimitiveRegistry) → []Mutation:

    // 1. Determine eligible primitives
    eligible = registry.All()
        .filter(p → p.Lifecycle() == Active)               // must be active
        .filter(p → tick.Value() - p.LastTick() >= p.Cadence().Value())  // cadence gating
        .filter(p → LayerStable(p.Layer(), snapshot))       // layer constraint
        .sortBy(p → p.Layer())                              // process lower layers first

    // 2. Match events to subscribers
    allMutations = []

    for each layer in 0..13:
        layerPrimitives = eligible.filter(p → p.Layer() == layer)

        // Primitives within the same layer can run in parallel
        results = parallel_map(layerPrimitives, fn(primitive):
            matchingEvents = events.filter(e →
                primitive.Subscriptions().any(pattern → pattern.Matches(e.Type))
            )

            // Invoke even with empty events (heartbeat/cadence invocation)
            return primitive.Process(tick, matchingEvents, snapshot)
        )

        // Collect mutations from this layer
        for result in results:
            match result:
                Ok(mutations) → allMutations.append(mutations)
                Err(error)    → emit error event, continue

        // Layer N must complete before Layer N+1 starts
        // (but within a layer, primitives run concurrently)

    return allMutations
```

## Layer Constraint

Layer N primitives activate only when Layer N-1 is **stable**. Stability means:

1. All primitives in the layer are in `Active` lifecycle state
2. The layer has produced consistent outputs (no state changes) for at least one full tick cycle
3. No primitives in the layer are in `Activating` or `Deactivating` states

```
fn LayerStable(layer Layer, snapshot Frozen<Snapshot>) → bool:
    if layer == 0:
        return true  // Layer 0 is always eligible

    lowerLayerPrimitives = snapshot.Primitives
        .filter(p → p.Layer == layer - 1)

    return lowerLayerPrimitives.all(p →
        p.Lifecycle == Active
        && p.LastTick > 0   // has been invoked at least once
    )
```

Within a single layer, primitives may execute concurrently — they share the same `Frozen<Snapshot>` and cannot affect each other's reads. Mutations are collected after all primitives in the layer complete.

---

## Cadence Gating

A primitive's `Cadence` controls how often it can be invoked. A cadence of 1 means every tick. A cadence of 5 means once every 5 ticks.

```
fn CadenceAllows(primitive Primitive, currentTick Tick) → bool:
    elapsed = currentTick.Value() - primitive.LastTick().Value()
    return elapsed >= primitive.Cadence().Value()
```

Cadence is a minimum interval, not a schedule. A primitive with cadence 5 runs *at most* once every 5 ticks, but may run less frequently if no matching events arrive and the primitive doesn't need heartbeat invocation.

## Mutation Application

Mutations from all waves are collected and applied atomically after the ripple loop completes. This is the Unit of Work pattern — either all mutations from a tick succeed, or none do.

```
fn ApplyMutations(mutations []Mutation, store Store, bus IBus, registry PrimitiveRegistry):
    for mutation in mutations:
        mutation.Accept(MutationApplier {
            VisitAddEvent(m):
                event = EventFactory.Create(m.type, m.source, m.content, m.causes, ...)
                store.Append(event)
                // Bus notifies subscribers automatically

            VisitAddEdge(m):
                event = EdgeFactory.Create(m.from, m.to, m.edgeType, m.weight, m.scope, ...)
                store.Append(event)

            VisitUpdateState(m):
                registry.UpdateState(m.primitiveID, m.key, m.value)

            VisitUpdateActivation(m):
                registry.UpdateActivation(m.primitiveID, m.level)

            VisitUpdateLifecycle(m):
                // Validates transition against state machine
                registry.UpdateLifecycle(m.primitiveID, m.state)
        })
```

### Mutation Validation

Before applying, mutations are validated:

- `AddEvent` — content must match `EventTypeRegistry` schema for the event type
- `AddEdge` — from/to actors must exist, weight must be valid `Weight`
- `UpdateLifecycle` — transition must be valid per `LifecycleState` state machine
- `UpdateActivation` — value must be valid `Activation` [0.0, 1.0]

Invalid mutations are rejected and an error event is emitted. Valid mutations from the same wave are still applied — one bad mutation doesn't roll back the entire wave.

## Subscription Matching

Events are matched to primitives via `SubscriptionPattern` — a glob pattern on `EventType`:

| Pattern | Matches |
|---|---|
| `trust.*` | `trust.updated`, `trust.score` |
| `actor.*` | `actor.registered`, `actor.suspended`, `actor.memorial` |
| `*` | everything |
| `clock.tick` | exactly `clock.tick` |
| `edge.*` | `edge.created`, `edge.superseded` |

Matching uses dot-separated segments. `*` matches one segment. `**` is not supported — subscribe to multiple patterns if you need broad matching.

## Concurrency Model

| Scope | Concurrency |
|---|---|
| Between ticks | Sequential — one tick at a time |
| Between layers in a wave | Sequential — Layer N completes before Layer N+1 |
| Within a layer in a wave | Parallel — primitives in the same layer run concurrently |
| Snapshot access | Safe — `Frozen<Snapshot>` is immutable, shared read-only |
| Mutation collection | Thread-safe — collected into a concurrent-safe list |
| Mutation application | Sequential — applied in order after wave completes |

The key invariant: **no primitive can observe another primitive's mutations within the same wave.** All primitives in a wave see the same frozen snapshot. Mutations become visible in the next wave.

## Tick Result

```
TickResult {
    Tick:       Tick
    Waves:      int                // how many waves before quiescence
    Mutations:  int                // total mutations applied
    Duration:   duration           // wall-clock time
    Quiesced:   bool               // true if terminated naturally, false if hit wave limit
}
```

## Error Handling

- **Primitive errors:** If `Process()` returns an error, the error is logged as an event (`primitive.error`). The primitive's activation is reduced. Other primitives continue.
- **Mutation validation errors:** Invalid mutations are rejected individually. An error event is emitted for each rejected mutation.
- **Store errors:** If the Store becomes unavailable during mutation application, the tick fails. Unapplied mutations are lost — they will be regenerated on the next tick from the same events.
- **Wave limit:** Not an error — it's a normal safety condition. The `TickResult.Quiesced` field indicates whether the system reached natural quiescence.

## Lifecycle Integration

Primitive lifecycle states interact with tick processing:

| State | Tick behaviour |
|---|---|
| `Dormant` | Not invoked. Ignored by the engine. |
| `Activating` | Not invoked. Waiting for activation to complete. |
| `Active` | Eligible for invocation (subject to cadence + layer). |
| `Processing` | Set by engine during `Process()` call. |
| `Emitting` | Set by engine during mutation application. |
| `Deactivating` | Not invoked. Waiting for deactivation to complete. |

The engine manages `Active → Processing → Emitting → Active` transitions automatically during tick processing. Primitives manage `Dormant → Activating → Active` and `Active → Deactivating → Dormant` via `UpdateLifecycle` mutations.
